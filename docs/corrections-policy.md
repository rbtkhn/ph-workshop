# Corrections Policy

Corrections are welcome. The goal is a useful public study corpus, not a frozen interpretation.

## What To Report

Open an issue or pull request for:

- incorrect title, date, URL, or source ID
- broken or changed source links
- transcript errors
- transcript body fidelity mismatches against the source lecture
- missing timestamps
- misread or overconfident annotations
- unclear separation between Jiang's claim and the annotator's interpretation
- rights or quotation concerns

## Correction Standard

A useful correction should include:

- the affected file or source ID
- the original source URL
- timestamp if video-derived
- the proposed correction
- why the current version is wrong, unclear, or incomplete

## Review Outcomes

Maintainers may:

- accept the correction
- ask for a source anchor
- narrow the wording
- move a disagreement into a counter-reading note
- mark the item as `needs_revision`
- defer the issue until source-text rights or transcript status is settled

Corrections should be recorded in `CHANGELOG.md` when they affect public-facing study material.
