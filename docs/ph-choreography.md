# The ph-civ / ph-apo / ph-mus Choreography

Predictive History now has three public motions through the same source body.

- `ph-civ` teaches the grammar: how historical forms, myths, institutions, literatures, empires, and pressures are made.
- `ph-apo` stress-tests the grammar: how those forms return under live pressure, escalation, finance, media, war, and sacred mobilization.
- `ph-mus` gives curated contact: artifacts, maps, texts, places, music, documents, diagrams, and institutions that let a reader encounter the pattern before argument hardens.

The choreography is not a content migration. The editorial source remains this repository. The public distribution layer is `rbtkhn/ph-civ`. The museum artifact archive remains outside Git in the local vault and shared cloud workspace.

## Reader Motion

A reader can enter from any surface:

- From `ph-civ`: begin with historical formation, then use `ph-mus` to make the pattern concrete, and finally use `ph-apo` where the pattern returns under pressure.
- From `ph-apo`: begin with crisis or forecast pressure, return through `ph-civ` for historical grounding, and use `ph-mus` to keep the analogy artifact-bound.
- From `ph-mus`: begin with an artifact, then route to the chapter card, transcript, and commentary before making any claim.

## Guardrails

- No modern pressure without historical grounding.
- No historical pattern without artifact humility.
- No museum artifact without a chapter route.
- No `ph-mus` route should imply local or cloud binary publication in Git.
- Apocalypse routes must keep date-sensitive and forecast-bearing claims visibly bounded.
